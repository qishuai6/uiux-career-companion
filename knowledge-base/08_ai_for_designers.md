# AI、AIGC 与设计师效率

## 核心判断

AI 能力不能只展示结果。面试官不关心“你会不会生成一张好看的图”，更关心你能不能用 AI 改变设计流程、解决真实问题。

AI 对设计师的价值：

- 提高探索效率。
- 快速做原型和 demo。
- 辅助设计规范 token 化。
- 自动化重复验收和整理。
- 帮助表达、复盘、研究和分析。

## AIGC 能力怎么放进作品集

不要只放 Midjourney、Stable Diffusion 或其他工具生成的成图。

要展示：

1. 需求拆解：你如何把模糊需求转成明确视觉方向。
2. Prompt 迭代：你如何逐步调试关键词、风格、构图、参数。
3. 方案筛选：你如何判断哪一版可用，哪一版不可用。
4. 项目落地：AI 结果如何进入真实设计稿、活动图、Banner、动效或组件。
5. 结果验证：上线数据、点击率、效率提升、交付周期缩短或团队反馈。

表达重点：

- 展示“AI 调教能力”，不是展示“AI 很厉害”。
- 展示“如何和项目结合”，不是展示“图很好看”。
- 展示“设计判断”，不是展示“工具熟练度”。

## AI Coding 对设计师的价值

AI Coding 的意义不是让设计师变成前端，而是让设计师能解决以前卡在设计和开发之间的灰色问题。

适合场景：

- 把设计规范转成 Token JSON。
- 做本地交互 demo。
- 做设计验收工具。
- 快速验证复杂动效和状态逻辑。
- 生成组件文档或规范检查脚本。
- 根据真实数据做可视化原型。

作品集表达可以这样写：

```text
我不是把 AI coding 当成炫技，而是用它解决设计协同中的具体问题：例如将 Figma 规范转成前端 Token、用本地 demo 验证交互手感、用检查脚本降低设计验收成本。
```

## 设计师使用 AI 的边界

AI 可以帮你：

- 快速发散方案。
- 整理资料。
- 生成初稿。
- 写脚本。
- 搭 demo。
- 做检查清单。

AI 不能替你：

- 定义真实问题。
- 判断业务优先级。
- 理解组织约束。
- 对结果负责。
- 和团队建立共识。

所以设计师的核心竞争力不是“会用哪个 AI 工具”，而是“能不能用 AI 放大自己的判断和交付能力”。

## AI 能力标签

简历或作品集中不要写：

- AI 熟练度 90%
- 精通 AI 绘图
- 会使用 Midjourney、ChatGPT

更好的写法：

- 使用 AI 辅助完成活动视觉探索，将创意探索周期从 X 天缩短到 Y 天。
- 用 AI coding 搭建本地交互 demo，提前验证复杂流程的状态和反馈。
- 将设计规范整理为 Token 结构，推动设计与前端样式对齐。
- 建立 AI 辅助验收 SOP，减少重复走查成本。

## 推荐输出

当用户问“设计师怎么用 AI”时，优先给场景清单：

- 创意探索
- 研究资料整理
- 访谈提纲和洞察提炼
- 文案和信息架构辅助
- 组件规范整理
- Token 化
- 交互 demo
- 设计验收
- 作品集表达

当用户问“AI 能力怎么写进作品集”时，优先给项目表达框架：

1. 原始问题。
2. 为什么需要 AI。
3. AI 参与哪个环节。
4. 人的设计判断在哪里。
5. 最终如何落地。
6. 带来什么效率、质量或结果变化。

# 用户研究与体验洞察

## 核心判断

用户研究不是为了让作品集显得完整，而是为了降低设计判断的不确定性。

研究前先问：

- 我们到底要知道什么？
- 这个问题是否适合用研究解决？
- 研究结论会影响什么设计决策？
- 如果研究结果和预期不同，我们会调整什么？

## 用户访谈准备

访谈前准备四件事：

1. 目的：想了解什么信息，服务哪个决策。
2. 对象：找谁访谈，为什么是这类人。
3. 假设：把功能价值、问题和场景口语化。
4. 提纲：按主题分组问题，而不是写一串审问式问题。

访谈对象招募可来自：

- 产品内招募
- 问卷筛选
- 公司内部相关角色
- 外部招募服务
- 客服、销售、运营推荐的典型用户

## 访谈原则

最近一次原则：

- 不要让用户直接总结观点。
- 让用户讲最近一次真实经历。
- 用户负责回忆，研究者负责提炼。

鹦鹉学舌原则：

- 遇到关键表述，用用户的话复述一遍。
- 再用自己的话总结，并请用户纠正。
- 目的是避免理解偏差。

沉默 10 秒原则：

- 提问后不要急着补充解释。
- 展示方案后给用户自己浏览和思考。
- 沉默常常能换来更细的回答。

不答反问原则：

- 用户问“这里会展示什么”时，不急着解释方案。
- 先问“你觉得这里会展示什么，为什么？”
- 这样能看出用户预期和心智模型。

转场衔接原则：

- 访谈不是照着提纲审问。
- 每个模块之间要做简短总结，再引出下一个任务。

## 问题设计

好的问题：

- “你最近一次使用这个功能是什么时候？”
- “当时你想完成什么？”
- “你是怎么判断下一步该点哪里？”
- “哪个地方让你犹豫？”
- “如果让你改一个地方，你会先改哪里？”

弱问题：

- “你喜欢这个设计吗？”
- “你觉得这个功能有用吗？”
- “你会不会使用？”
- “这样是不是更高效？”

弱问题会诱导用户给态度，强问题会逼近行为和场景。

## 洞察提炼

访谈结束后不要只整理语录，要转成：

- 行为事实：用户实际做了什么。
- 场景条件：什么时候、为什么发生。
- 阻碍因素：卡在哪里。
- 情绪信号：焦虑、犹豫、爽点、信任。
- 设计机会：可以通过什么设计动作降低阻碍。

洞察表达模板：

```text
对于【用户类型】来说，在【具体场景】下，他们想完成【目标】，但因为【阻碍】，导致【结果】。因此设计机会是【可行动方向】。
```

## 用户分层

用户分层不是为了画画像，而是为了服务差异化设计。

可按这些维度分：

- 行为频率：高频、低频、新用户、沉默用户。
- 目标差异：效率型、探索型、决策型、管理型。
- 能力差异：新手、熟练者、专家。
- 价值差异：高价值用户、潜力用户、普通用户。
- 场景差异：移动场景、桌面场景、协作场景、审批场景。

分层后要回答：

- 哪类用户优先级最高？
- 哪类用户当前体验最差？
- 哪类用户最影响业务目标？
- 设计方案是服务主用户，还是兼容边缘用户？

## 问卷使用

问卷适合验证范围和比例，不适合深挖原因。

适合问卷：

- 选择偏好
- 满意度
- 功能使用频率
- 问题分布
- 人群筛选

不适合只靠问卷：

- 深层动机
- 复杂决策过程
- 用户为什么这样理解界面
- 方案为什么让用户困惑

问卷前最好先用访谈或客服反馈形成假设，再用问卷验证规模。

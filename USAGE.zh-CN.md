# UI/UX Career Companion 使用方法

这是一套给 Codex 使用的 UI/UX 职业陪伴 skill。它不是通用设计百科，而是会优先检索你的本地 UI/UX 知识库；如果没有配置知识库，也会基于 skill 内置的诊断框架输出判断、框架、清单或行动建议。

## 适合用来做什么

你可以在这些场景里调用它：

- 作品集诊断：看项目叙事、页面结构、表达重点是否清楚。
- 面试准备：拆解面试题，整理回答结构和表达策略。
- 跳槽判断：分析岗位、公司、作品集准备度和机会风险。
- 项目介绍框架：把零散项目经历整理成作品集 case study。
- 用户研究方法：访谈、问卷、用户分层、洞察提炼。
- 交互设计：流程设计、信息架构、表单、列表、详情页设计。
- UI 和设计系统：视觉规范、Design Token、组件体系。
- 体验度量：如何定义指标、验证设计效果、讲清楚设计价值。
- AI 与设计效率：设计师如何使用 AI、如何把 AI 能力写进作品集。

## 如何触发

在 Codex 里直接提出 UI/UX 相关问题即可。只要问题明显属于作品集、面试、设计方法、交互、UI、设计系统、体验度量或 AI 设计效率，Codex 会自动使用这个 skill。

也可以在问题里明确写：

```text
用 uiux-career-companion 帮我看一下这个作品集项目
```

或者：

```text
用 UI/UX Career Companion 帮我整理这个项目的 case study 结构
```

## 推荐提问方式

### 作品集诊断

```text
帮我看这个作品集项目的问题：
项目背景是……
我负责……
现在页面结构是……
我担心的问题是……
```

它会优先判断：

- 项目一句话是否清楚
- 问题定义是否成立
- 你的角色是否有说服力
- 设计过程是否像真实工作
- 结果和验证是否能支撑作品集表达

### 项目介绍框架

```text
我输入作品集项目大概内容，你帮我作项目介绍的框架：
项目类型：
业务背景：
目标用户：
核心问题：
我的角色：
关键动作：
结果：
```

它会输出适合放进作品集或面试讲述的结构，通常包括：

- 项目一句话定义
- 项目背景
- 核心问题
- 我的角色
- 设计判断
- 方案动作
- 结果与验证
- 面试讲述版本

### 面试题准备

```text
面试官问：你如何证明这个设计有效？
请用 uiux-career-companion 帮我整理回答。
```

它会帮你把回答从“讲流程”改成“讲判断依据”，更适合真实面试。

### 跳槽和岗位判断

```text
我现在想投这个 UI/UX 岗位，JD 是……
我的背景是……
帮我判断匹配度和准备重点。
```

它会从作品集、经验匹配、岗位风险、面试准备几个角度给建议。

## 本地知识库配置

本地知识库不是强制要求，但建议配置。推荐设置环境变量：

```bash
export UIUX_KB_ROOT="/absolute/path/to/UIUX知识库_整合版"
```

这个 skill 也自带一份精选版 `knowledge-base`，所以别人安装后不配置外部知识库也能直接用。你也可以把自己的知识库放在 skill 目录下，命名为：

```bash
knowledge-base
```

搜索脚本位置：

```bash
scripts/search_kb.sh
```

脚本会按这个顺序找知识库：

1. `UIUX_KB_ROOT`
2. `$HOME/Desktop/UIUX知识库_整合版` 等常见本地目录
3. skill 目录下的内置 `knowledge-base`

如果没有找到任何知识库，skill 不应该中断回答，而是说明“我没有检索到知识库，下面基于 skill 内置框架判断。”

## 手动测试搜索脚本

可以直接在终端运行：

```bash
scripts/search_kb.sh "作品集 面试"
```

也可以限定搜索某个知识库分区：

```bash
scripts/search_kb.sh "作品集 面试" "01_求职面试与作品集"
```

如果返回相关 Markdown 文件和行号，说明知识库路径和搜索脚本正常。默认情况下，新用户会搜到内置精选库；配置 `UIUX_KB_ROOT` 后会优先搜自己的知识库。

## 输出风格

这个 skill 的回答应该像一个有经验的 UI/UX 职业顾问，而不是资料搬运工。

它应该优先给：

- 诊断
- 优先级
- 修改方向
- 面试表达框架
- 作品集结构
- 决策建议

它不应该只说：

- “可以参考这些方法”
- “建议多看看资料”
- “UI/UX 很重要”

## 注意事项

- 这个仓库只包含 skill 本身，不包含你的本地 UI/UX 知识库内容。
- GitHub 上的仓库是 private，适合继续放个人工作流。
- 如果未来要公开仓库，不需要暴露个人知识库路径；让用户通过 `UIUX_KB_ROOT` 或 `knowledge-base` 自行配置即可。
